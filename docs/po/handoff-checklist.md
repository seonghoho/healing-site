# 역할별 핸드오프 체크리스트

## UX
- Must 범위 기준으로 핵심 여정 1개(`/` → `/today` → 완료 → `/history`) 우선 설계
- `/today`의 "1개 미션 집중 + 즉시 피드백" 인터랙션 먼저 검증
- `/history`는 최근 30일 고정 정책 전제로 IA 확정

## Design
- 모바일 우선(360px) 홈/오늘/기록 공통 패턴 설계
- 완료 직후 칭찬 메시지의 시각적 우선순위 강화
- 접근성 기본 요건(포커스, 라벨, 대비) 시안 단계 점검

## FE
- Must 기능 선구현: 랜딩 CTA, 오늘 미션 완료, 30일 히스토리, localStorage 유지
- KPI 측정 가능한 QA 시나리오 먼저 작성
- Should/Could는 Must 완료 후 순차 반영
