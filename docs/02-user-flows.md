# 02. User Flows

## 플로우 A: 번아웃 사용자
1. 사용자가 랜딩(`/`)에 진입한다.
2. 60초 호흡 섹션에서 "들이쉬기/내쉬기" 가이드를 본다.
3. "시작하기"로 `/today`에 이동한다.
4. 오늘의 미션 1개를 확인한다.
5. 완료 버튼을 누르면 칭찬 메시지와 누적 완료 수가 즉시 갱신된다.
6. `/history`에서 오늘 기록이 반영된 것을 확인한다.

## 플로우 B: 일반 사용자
1. 사용자가 `/today`를 직접 방문한다.
2. 오늘의 미션을 수행 후 완료 체크한다.
3. `/history`에서 최근 기록을 확인한다.
4. 다음 방문 시 같은 날에는 중복 완료가 증가하지 않는다.

## 예외 케이스
- 저장소 접근 실패(localStorage 비활성): 메모리 기본 상태로 동작하고 오류 없이 UI 유지
- JSON 파싱 실패(손상 데이터): 기본 상태로 초기화 후 계속 사용
- 같은 날 중복 완료 시도: 누적 카운트 증가 없이 안내 메시지 표시
- 기록이 없는 상태에서 `/history` 진입: 최근 30일 모두 미완료 상태로 표시
