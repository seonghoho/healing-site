# 02. User Flows

## 플로우 A: 번아웃 사용자(기본 여정)

### A-1. 최초 진입 안정화 (`/`)
1. 사용자가 랜딩(`/`)에 진입한다.
2. 호흡 카드는 `Idle` 상태로 "시작" 액션만 먼저 보여준다.
3. 사용자가 시작하면 60초 타이머와 "들이쉬기/내쉬기" 가이드가 `Running` 상태로 전환된다.
4. 60초가 끝나면 `Done` 상태로 전환되고 `/today` 이동 CTA가 노출된다.

### A-2. 최소 행동 완료 (`/today`)
1. 오늘의 미션 1개만 노출해 선택 부담을 줄인다.
2. 사용자가 완료 버튼을 누르면 당일 완료가 기록되고 칭찬 메시지(`aria-live`)가 출력된다.
3. 누적 완료 수를 즉시 갱신한다.

### A-3. 기록 확인 (`/history`)
1. 최근 30일 리스트에서 오늘 상태를 확인한다.
2. 완료된 날짜는 완료 배지와 미션명을 함께 표시한다.
3. 미완료 날짜는 빈 상태가 아니라 명시적 "미완료"로 표시한다.

## 플로우 B: 일반 사용자(직접 진입)
1. 사용자가 `/today`를 직접 방문한다.
2. 오늘의 미션을 수행하고 완료 버튼을 누른다.
3. `/history`에서 반영 결과를 확인한다.
4. 같은 날 재완료를 시도하면 중복 완료로 처리되고 카운트는 증가하지 않는다.

## 예외/회복 플로우

### E-1. 의욕 0 상태
1. 사용자가 호흡 시작이나 미션 수행을 망설인다.
2. UI는 오류 없이 현재 상태를 유지하고 강제 입력을 요구하지 않는다.
3. 사용자는 언제든 다시 시작하거나 다른 라우트로 이동할 수 있다.

### E-2. 미완료 이탈
1. 사용자가 호흡 도중 또는 미션 수행 전 이탈한다.
2. 완료 이벤트가 발생하지 않았으므로 기록은 증가하지 않는다.
3. 다음 진입 시 동일한 오늘 미션으로 재시도한다.

### E-3. 같은 날 중복 완료 시도
1. 이미 완료된 날에 완료 버튼을 다시 누른다.
2. 시스템은 멱등 처리로 카운트를 유지한다.
3. UI는 "이미 오늘 완료했어요" 메시지만 갱신한다.

### E-4. 리셋
1. 사용자가 개발용/관리용 리셋 액션을 실행한다.
2. 당일 완료 여부와 누적 카운트를 기본값으로 재설정한다.
3. 이후 즉시 다시 완료 처리가 가능하다.

### E-5. 저장소 접근 실패 (`localStorage` 비활성)
1. 영속 저장에 실패하면 메모리 상태로 폴백한다.
2. 핵심 여정(호흡/완료/조회)은 계속 동작한다.
3. 새로고침 시 기록이 유지되지 않을 수 있음을 비차단 방식으로 안내한다.

### E-6. JSON 파싱 실패 (손상 데이터)
1. 저장된 문자열 파싱에서 오류가 발생한다.
2. 앱은 기본 상태로 안전하게 재초기화한다.
3. 사용자 플로우는 중단되지 않고 즉시 계속 진행된다.
