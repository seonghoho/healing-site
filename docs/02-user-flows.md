# 02. User Flows

## 공통 원칙
- 목표는 "완벽한 수행"이 아니라 "오늘 1회 성공 경험"이다.
- 모든 플로우는 1분 안에 끝낼 수 있는 행동 1개를 기본 성공으로 인정한다.
- 미완료는 실패가 아니라 "다음 시도 대기" 상태로 처리한다.

## 플로우 A: 번아웃 모드 (초저강도)
### 목적
- 의욕 0 사용자도 실패하지 않고 오늘을 마무리한다.

### 진입
- `/today`에서 `오늘은 쉬운 걸로` 토글을 켠다.

### 1분 내 성공 행동 1개
- `1분 미션 완료` 버튼 1회 누르기

### 단계
1. 사용자가 `/today`에 진입한다.
2. `오늘은 쉬운 걸로` 토글을 ON으로 전환한다.
3. 시스템은 초저강도 단계로 `30초 호흡` 카드를 먼저 노출한다.
4. 30초 호흡이 끝나면 `1분 미션` 카드(행동 1개)로 자동 전환한다.
5. 사용자가 `1분 미션 완료` 버튼을 1회 누른다.
6. 시스템은 "오늘 할 일 완료" 피드백을 노출하고 `/history` 확인 CTA를 제공한다.

### 예외 분기
- 의욕 0(행동 시작 어려움): 토글 ON 즉시 가장 쉬운 30초 호흡만 먼저 제공한다.
- 미완료: `1분 미션`을 당일 유지하고 "나중에 1분만" 재시도 CTA를 제공한다.
- 다음날: 번아웃 모드는 OFF 기본값으로 시작하되, 사용자가 다시 ON으로 즉시 전환할 수 있다.
- 리셋: 사용자가 리셋을 실행하면 번아웃 모드 상태/오늘 기록/히스토리를 초기화한다.
- 기록 없음: `/history`는 빈 상태 메시지와 `/today` 복귀 CTA를 노출한다.

## 플로우 B: 일반 모드 (기본)
### 목적
- 일반 사용자가 기존 루틴으로 오늘 미션을 완료한다.

### 진입
- 사용자가 `/today`를 직접 방문한다. (번아웃 모드 OFF)

### 1분 내 성공 행동 1개
- `완료 체크` 버튼 1회 누르기

### 단계
1. 사용자가 오늘 미션 카드(기본 모드)를 확인한다.
2. `완료 체크`를 1회 누른다.
3. 시스템은 완료 배지/칭찬 메시지/누적 수치를 노출한다.
4. 사용자는 `/history`에서 오늘 상태를 확인한다.

### 예외 분기
- 미완료: 페이지 이탈 후 재진입 시 같은 미션을 다시 보여준다.
- 다음날: 새 날짜의 미션 상태는 기본(미완료)으로 시작한다.
- 리셋: 확인 후 리셋 시 오늘/과거 기록을 초기화한다.

## 완료 판정 기준
- 번아웃 모드: `1분 미션 완료` 1회면 당일 성공으로 기록한다.
- 일반 모드: `완료 체크` 1회면 당일 성공으로 기록한다.
