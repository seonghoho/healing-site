# 02. User Flows

## 공통 원칙
- 목표는 "완벽한 수행"이 아니라 "오늘 1회 성공 경험"이다.
- 모든 플로우는 1분 안에 끝낼 수 있는 행동 1개를 기본 성공으로 인정한다.
- 미완료는 실패가 아니라 "다음 시도 대기" 상태로 처리한다.

## 플로우 A: 번아웃 사용자 (의욕 저하)
### 진입 조건
- 사용자가 `/`에서 시작하거나, 해야 할 일을 떠올리는 것만으로도 부담을 느끼는 상태

### 1분 내 성공 행동 1개
- `호흡 시작` 버튼 1회 누르기 (60초를 끝내지 못해도 성공으로 기록하지 않고, 행동 자체를 성공으로 안내)

### 단계
1. `/` 진입 시 "지금은 하나만 해도 충분" 메시지를 먼저 노출한다.
2. 사용자는 호흡 카드에서 `호흡 시작` 1회만 누른다.
3. 시스템은 즉시 `/today`로 이동 가능한 CTA를 보여준다.
4. `/today`에서 "오늘 한 가지" 미션을 제안한다.
5. 사용자가 완료 체크 시 당일 완료로 기록하고 칭찬 메시지를 노출한다.
6. `/history`에서 오늘 상태를 확인한다.

### 예외 분기
- 의욕 0: `/today`에서 기본 미션을 "60초 숨 고르기"로 단순화하고 완료 버튼 하나만 제공한다.
- 미완료: 당일 미완료로 남기되 죄책감 유발 문구 없이 "내일 다시 1분" 안내를 노출한다.
- 다음날: 전날 미완료는 그대로 보존하고 오늘 카드만 새로 시작 상태로 제공한다.
- 리셋: 사용자가 리셋을 실행하면 오늘/기록 상태를 초기화하고 랜딩(`/`)으로 복귀시킨다.
- 기록 없음: `/history`에서 빈 상태 메시지와 `/today` 이동 CTA를 노출한다.

## 플로우 B: 일반 사용자 (루틴 유지)
### 진입 조건
- 사용자가 `/today`를 직접 열어 오늘 미션을 처리하려는 상태

### 1분 내 성공 행동 1개
- `완료 체크` 버튼 1회 누르기

### 단계
1. 사용자가 `/today`에 진입한다.
2. 오늘 미션 제목/설명을 확인한다.
3. `완료 체크` 1회로 당일 미션을 완료한다.
4. 시스템은 완료 상태와 칭찬 메시지를 즉시 노출한다.
5. 사용자는 `/history`로 이동해 최근 누적을 확인한다.

### 예외 분기
- 미완료: 페이지를 이탈해도 상태는 미완료로 유지되고 재진입 시 같은 미션을 다시 보여준다.
- 다음날: 당일 완료 여부와 무관하게 새 날짜의 미션 상태는 기본(미완료)으로 시작한다.
- 리셋: 리셋 시 연속성 데이터도 함께 초기화되며, 확인 후에만 실행된다.

## 완료 판정 기준
- 번아웃 사용자: 행동 시작(버튼 1회)과 미션 완료를 분리해 진입 장벽을 낮춘다.
- 일반 사용자: 당일 완료 체크 1회로 기록/히스토리 반영까지 끝난다.
