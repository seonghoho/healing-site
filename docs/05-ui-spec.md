# 05. UI Spec (Structure-Focused)

## 공통 구조
- 상단: 브랜드 + 글로벌 내비게이션(홈/오늘/기록)
- 본문: `Eyebrow` -> `H1` -> 보조 설명 -> 핵심 카드 1개
- 하단: 다음 액션 CTA 또는 상태 안내
- 레이아웃: 모바일 우선 단일 컬럼, 카드 중심 정보 밀도 유지

## `/` 랜딩
### 정보 구조
1. 진입 맥락 문구(Eyebrow)
2. 60초 호흡 제안(H1)
3. 부담 완화 설명
4. 호흡 카드
5. `/today` 이동 CTA

### 호흡 카드 구성
- 가이드 텍스트: 들이쉬기/내쉬기
- 시각 피드백: 원형 애니메이션 + 진행률/남은 시간
- 제어: 시작, 다시하기

### 상태 모델
- `Idle`: 시작 버튼만 강조, 진행률 숨김
- `Running`: 남은 시간/가이드 문구 실시간 갱신
- `Done`: 완료 메시지와 다음 라우트 CTA 노출

## `/today`
### 정보 구조
1. 오늘의 미션 안내(Eyebrow + H1)
2. "한 가지 미션" 원칙 설명
3. 미션 카드
4. 상태 피드백 영역
5. 개발용 리셋 액션(배포 환경 비노출 가능)

### 미션 카드 구성
- 미션 제목/설명
- 예상 소요 시간
- 완료 버튼
- 칭찬 메시지(`aria-live`)
- 누적 완료 수치

### 상태 모델
- `NotCompleted`: 완료 버튼 활성, 기본 안내 문구
- `Completed`: 칭찬 메시지 + 누적 수 갱신 + 완료 배지
- `DuplicateAttempt`: 카운트 고정, 중복 완료 안내 메시지 출력

## `/history`
### 정보 구조
1. 회복 기록 안내(Eyebrow + H1)
2. 최근 30일 설명
3. 기록 리스트 카드

### 기록 리스트 구성
- 날짜
- 완료/미완료 배지
- 완료일일 때 미션 제목 표시

### 상태 모델
- `HasRecords`: 완료/미완료 혼합 표시
- `EmptyOrReset`: 최근 30일을 기본 미완료로 렌더링

## 공통 예외 처리 UI 원칙
- 저장소 실패: 플로우를 차단하지 않고 메모리 폴백으로 계속 진행
- 파싱 실패: 기본 상태 재초기화 후 즉시 화면 복구
- 미완료 이탈 복귀: 사용자가 이전 맥락을 이해할 수 있는 기본 안내 문구 유지
