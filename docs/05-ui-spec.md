# 05. UI Spec (Text Wireframe)

## 공통 카피 원칙
- 페이지 제목은 1문장, 설명은 1~2문장으로 유지
- 버튼 문구는 동사로 시작하고 2~8자 범위 권장
- 실패 문구는 원인 + 재시도 행동을 함께 제공
- 빈 상태 문구는 "시작 행동 1개"만 제안

## `/` (랜딩)

### 헤더/본문
- Eyebrow: `오늘도 천천히 회복해요`
- 제목(H1): `60초만, 숨을 고르며 시작해요`
- 설명: `길게 하지 않아도 괜찮아요. 지금 1분만 함께 호흡해요.`

### 호흡 카드
- 상태 라벨(들이쉬기): `들이쉬어요`
- 상태 라벨(내쉬기): `내쉬어요`
- 진행 라벨: `남은 시간`
- 완료 문구: `호흡 1회 완료. 몸의 긴장을 조금 내려놨어요.`
- 빈 상태 문구: `아직 시작 전이에요. 준비되면 시작해요.`
- 실패 문구: `타이머를 불러오지 못했어요. 다시 시작해볼까요?`

### 버튼
- 기본 버튼: `호흡 시작`
- 진행 중 버튼: `잠시 멈추기`
- 재시작 버튼: `다시 시작`
- CTA 버튼: `오늘 미션 보기` (이동: `/today`)

## `/today`

### 헤더/본문
- Eyebrow: `오늘의 회복`
- 제목(H1): `오늘은 한 가지만 해요`
- 설명: `부담 없는 미션 하나면 충분해요. 끝나면 바로 기록할 수 있어요.`

### 미션 카드
- 카드 제목: `오늘의 미션`
- 예상 시간 라벨: `예상 1~3분`
- 완료 수 라벨: `지금까지 완료`
- 완료 문구(aria-live): `좋아요. 오늘의 미션을 기록했어요.`
- 미완료 안내: `아직 완료 전이에요. 가능할 때 체크해요.`
- 빈 상태 문구: `아직 선택된 미션이 없어요. 미션을 불러와요.`
- 실패 문구: `미션을 불러오지 못했어요. 잠시 후 다시 시도해요.`

### 버튼
- 미션 생성: `미션 받기`
- 완료 체크: `완료했어요`
- 다음 미션: `다른 미션 보기`
- 재시도: `다시 시도`
- 개발 전용 초기화: `기록 초기화`

## `/history`

### 헤더/본문
- Eyebrow: `회복 기록`
- 제목(H1): `최근 30일의 작은 성취`
- 설명: `완료한 날만 천천히 쌓아가요. 비어 있는 날이 있어도 괜찮아요.`

### 기록 리스트/카드
- 리스트 제목: `기록 내역`
- 완료 배지: `완료`
- 미완료 배지: `미완료`
- 항목 보조 문구(완료): `미션: {missionTitle}`
- 항목 보조 문구(미완료): `기록된 미션이 없어요`
- 빈 상태 문구: `아직 기록이 없어요. 오늘의 미션부터 시작해요.`
- 실패 문구: `기록을 불러오지 못했어요. 다시 시도해요.`

### 버튼
- 새로고침: `다시 불러오기`
- 오늘 페이지 이동: `오늘 미션으로`
