# 05. UI Spec (Structure & States)

## 공통 구조
- 헤더: 브랜드 + 고정 내비게이션(`홈 / 오늘 / 기록`)
- 본문: 제목, 짧은 설명, 핵심 카드 1개, 보조 안내 1개
- 행동 규칙: 한 화면에서 우선 CTA는 1개만 강조

## `/` 랜딩 상태
| 상태 | 표시 구조 | 사용자 행동 1개 |
| --- | --- | --- |
| 기본 | 호흡 소개 문구 + 60초 호흡 카드 + `시작하기` CTA | `시작하기` |
| 완료 | 호흡 진행 100% + 완료 안내 + `/today` 이동 CTA | `오늘 미션 보기` |
| 빈 | 개인화 데이터 없음, 기본 소개 문구만 노출 | `시작하기` |
| 실패 | 타이머/저장 실패 안내 + 텍스트 호흡 가이드 | `다시 시도` |

## `/today` 상태
| 상태 | 표시 구조 | 사용자 행동 1개 |
| --- | --- | --- |
| 기본 | 모드 토글 + 오늘 미션 카드 + 완료 버튼 | `완료 체크` |
| 완료 | 완료 배지 + 피드백 메시지 + 누적 수치 | `기록 보러 가기` |
| 빈 | 미션 데이터 없음 안내 + 기본 미션 제안 | `기본 미션 받기` |
| 실패 | 저장/불러오기 실패 메시지 + 재시도 버튼 | `다시 시도` |

## `/today` 번아웃 모드 ON/OFF 규칙
| 모드 | 문구 규칙 | 버튼 규칙 | 미션 규칙 |
| --- | --- | --- | --- |
| OFF (기본) | 일반 루틴 안내 문구 노출 | `완료 체크` 버튼 1개 강조 | 기본 미션 1개 제공 |
| ON (번아웃) | 압박 완화 문구(예: "오늘은 쉬운 걸로") 노출 | `30초 호흡 시작` 후 `1분 미션 완료` 버튼만 순차 노출 | `30초 호흡 -> 1분 미션` 초저강도 단일 경로 적용 |

## `/history` 상태
| 상태 | 표시 구조 | 사용자 행동 1개 |
| --- | --- | --- |
| 기본 | 최근 30일 목록 + 날짜별 완료/미완료 배지 | `최근 1일 확인` |
| 완료 | 오늘 완료 항목 강조 + 연속 수행 정보 | `오늘 기록 확인` |
| 빈 | 기록 없음 메시지 + `/today` 이동 CTA | `오늘 미션 시작` |
| 실패 | 기록 로드 실패 메시지 + 새로고침 버튼 | `새로고침` |

## 예외 상태 반영 규칙
- 의욕 0: `/today` 번아웃 모드 ON 시 30초 호흡 단계를 먼저 고정 노출한다.
- 미완료: 번아웃/일반 모드 모두 미완료를 실패로 표기하지 않고 재시도 CTA를 제공한다.
- 다음날: `/history`에는 전날 상태를 보존하고 `/today`는 새 날짜 기본 상태로 시작한다.
- 리셋: 리셋 완료 시 모드/오늘 상태/기록 상태를 초기화한다.
- 기록 없음: `/history` 빈 상태를 오류와 구분해 명확히 표시한다.
