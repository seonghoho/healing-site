# 03. IA & Sitemap

## 현재 IA
- `/`: 진입 장벽을 낮추는 랜딩 + 호흡 시작
- `/today`: 오늘의 미션 1개 수행
- `/history`: 최근 기록 확인

## 라우트 표 (확정)
| Route | 화면명 | 핵심 목적 | 주요 사용자 행동 1개 | 주요 상태 |
| --- | --- | --- | --- | --- |
| `/` | 랜딩 | 부담 없이 시작하게 만들기 | `호흡 시작` 1회 누르기 | 기본 / 완료 / 실패 |
| `/today` | 오늘의 미션 | 오늘 미션 1개 완료 | `완료 체크` 1회 누르기 | 기본 / 완료 / 빈 / 실패 |
| `/history` | 기록 | 최근 흐름 확인 및 재시작 유도 | 최근 1일 상태 확인 | 기본 / 완료 / 빈 / 실패 |

## 확장 후보 (차기)
| Route | 목적 | 추가 시점 기준 |
| --- | --- | --- |
| `/onboarding` | 첫 방문자에게 30초 사용법 제공 | 이탈률이 랜딩에서 높을 때 |
| `/settings` | 알림 시간/문구 톤/데이터 초기화 관리 | 개인화 요구가 증가할 때 |
| `/insights` | 주간 회복 패턴 요약 제공 | 기록 데이터가 누적된 이후 |

## 내비게이션 원칙
- 상단 내비게이션은 항상 `홈 / 오늘 / 기록` 3개만 고정한다.
- 확장 후보 라우트는 기본 내비게이션이 아니라 보조 진입(설정 버튼/배너)으로 시작한다.
- 어느 화면에서도 다음 행동은 1개만 우선순위로 제시한다.
