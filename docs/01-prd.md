# 01. PRD

## MVP 목표
- 사용자가 1분 내로 `오늘의 미션 확인 -> 완료 -> 성취 피드백 확인`까지 끝낼 수 있게 한다.

## MVP 범위 (MoSCoW)

### Must
- `/` 랜딩에서 60초 호흡 가이드와 시작 CTA를 제공한다.
- `/today`에서 오늘의 미션 1개, 완료 체크, 칭찬 메시지, 누적 카운트를 제공한다.
- `/history`에서 최근 30일의 날짜별 완료 여부를 확인할 수 있다.
- 로그인 없이 `localStorage`에 완료 기록을 저장하고 새로고침 후에도 유지한다.
- 홈/오늘/기록 네비게이션을 포함한 기본 레이아웃을 제공한다.

### Should
- 날짜 기반 deterministic 미션 선택(같은 날 재진입 시 동일 미션)을 적용한다.
- 개발/운영 점검용 `resetAll()` 기능을 제공한다.
- 문서(`docs`)와 구현 간 매핑을 릴리즈 기준으로 유지한다.

### Could
- 주간 통계(연속 기록 일수)를 추가한다.
- 미션 난이도/카테고리 필터를 제공한다.
- 다국어 지원을 제공한다.

## 성공 지표 (MVP, 2~3개)
1. 핵심 여정 완료율 95% 이상: `/ -> /today -> 완료 -> /history`를 오류 없이 수행한다.
2. 저장 유지율 100%: 당일 완료 후 새로고침/재진입 시 완료 상태가 유지된다.
3. 즉시 피드백 노출률 100%: 완료 클릭 후 1초 이내 칭찬 메시지 또는 상태 변화가 표시된다.

## 비기능 요구사항

### 모바일
- 360px 기준에서 주요 텍스트/버튼이 가로 스크롤 없이 보인다.
- 터치 대상(버튼/체크)은 최소 44x44px 기준을 만족한다.

### 접근성
- 모든 인터랙션 요소는 키보드로 접근 가능하고 포커스가 시각적으로 표시된다.
- 링크/버튼/입력 요소에 접근 가능한 이름(`aria-label` 또는 텍스트 라벨)을 제공한다.
- 본문 텍스트와 배경의 명도 대비는 WCAG AA 기준을 충족한다.
