# 01. PRD

## 목표
- MVP에서 사용자가 **1분 내로 오늘의 미션 확인 → 완료 기록 → 성취 피드백 확인**까지 마칠 수 있게 한다.

## MVP 범위 (확정)

### Must (MVP 출시 필수)
- `/` 랜딩: 60초 호흡 가이드와 시작 CTA
- `/today`: 오늘의 미션 1개 표시, 완료 체크, 칭찬 메시지, 누적 카운트
- `/history`: 최근 30일 기록 확인(날짜/완료 여부)
- 로그인 없이 `localStorage`에 데이터 저장 및 새로고침 유지
- 기본 레이아웃(홈/오늘/기록 네비게이션)
- 접근성 기본(포커스, aria-label, 텍스트 대비)

### Should (MVP 직후 1차 고도화)
- 날짜 기반 deterministic 미션 선택(같은 날 동일 미션)
- 개발/운영 점검용 `resetAll()` 제공
- 문서-구현 매핑 유지(docs를 단일 진실원으로 관리)

### Could (백로그)
- 주간 통계(연속 기록 일수)
- 미션 난이도/카테고리 필터
- 다국어 지원

## 성공 지표 (MVP, 2~3개)
1. **핵심 여정 완료율 95%+**: `/ -> /today -> 완료 -> /history` 시나리오를 오류 없이 수행
2. **저장 유지율 100%**: 당일 완료 후 새로고침/재진입 시 완료 상태가 유지
3. **즉시 피드백 노출률 100%**: 완료 클릭 직후 칭찬 메시지/상태 변화가 즉시 표시

## 비기능 요구사항
- 모바일 우선 레이아웃(360px 기준 가독성 확보)
- 접근성: 키보드 포커스 표시, 링크/버튼 이름 제공, 명도 대비 확보
- 성능: 주요 페이지 첫 렌더 시 과도한 스크립트 의존 최소화
